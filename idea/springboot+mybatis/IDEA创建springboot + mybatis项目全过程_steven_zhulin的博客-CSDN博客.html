IDEA创建springboot + mybatis项目全过程

stephenChen946 2019-06-02 12:42:12 79486 收藏 386

鉴于隔很久再在IDEA新建springboot项目时，会出现对步骤不确定的情况，因此，写下这篇博客记录创建一个可运行的springboot+mybatis项目的全过程。

步骤如下：

1.打开IDEA

2.File  ==>  new  ==>  project  ,如图：

3.选择spring Initializr ==> 右边的Project SDK我选的是我已经安装的1.8版本,其他默认 ==> 点击next

4.填写Group （自己随意就行，我的是cn + 个人英文名 + study） ==> 填写 Artifact （也是自己随意就行，这个也是你的项目名） ==> 点击next   ，如图：

5.选择项目所需依赖（由于项目是springboot+mybatis的Java后台项目，因此需要勾上三个依赖） ==>  点击next  ，如图：

6.选择项目路径  ==> 点击finish ,如图：

6.最终生成的项目代码目录如图（resources目录下的static和templates目录无用，可以删掉）：

（注：如果DemoprojectApplication类(即启动类)的图标显示是 J 而不是 C ，则说名该项目还没有被初始化为maven项目，

只需要在项目的pom.xml文件上右键，再选择 Add as maven 即可）

至此，springboot项目搭建完成一半，还需要做mybatis的配置，如下：

7.数据库和mybatis的配置

7.1 在application.properties填入数据库连接参数和mybatis配置信息，application.properties内容如下： 

  1. #1.项目启动的端口
  2. server.port=18902
  3.  
  4. #2.数据库连接参数
  5. #2.1jdbc驱动，示数据库厂商决定，这是mysql的驱动
  6. jdbc.driver=com.mysql.cj.jdbc.Driver
  7. #2.2数据库连接url，包括ip(127.0.0.1)、端口(3306)、数据库名(testdb)
  8. jdbc.url=jdbc:mysql://127.0.0.1:3306/testdb?useUnicode=true&characterEncoding=utf-8&useSSL=false
  9. #2.3数据库账号名
  10. jdbc.username=root
  11. #2.4数据库密码
  12. jdbc.password=mypassword
  13.  
  14. #3.Mybatis配置
  15. #3.1 mybatis配置文件所在路径
  16. mybatis_config_file=mybatis-config.xml
  17. #3.2 mapper文件所在路径，这样写可匹配mapper目录下的所有mapper，包括其子目录下的
  18. mapper_path=/mapper/**/**.xml
  19. #3.3 entity所在包
  20. entity_package=cn.stephen.study.demoproject.entity

示例如图：

7.2 在resources目录下新建 mybatis-config.xml文件，文件内容如下：

  1. <?xml version="1.0" encoding="UTF-8" ?>
  2. <!DOCTYPE configuration
  3.         PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
  4.         "http://mybatis.org/dtd/mybatis-3-config.dtd">
  5. <!-- 配置文件的根元素 -->
  6. <configuration>
  7.     <!--配置全局属性-->
  8.     <settings>
  9.         <!--使用jdbc的getGeneratedKeys获取数据库自增主键值-->
  10.         <setting name="useGeneratedKeys" value="true"/>
  11.         <!--使用列标签替换列别名 默认未true-->
  12.         <setting name="useColumnLabel" value="true" />
  13.         <!--开启驼峰式命名转换：Table{create_time} -> Entity{createTime}-->
  14.         <setting name="mapUnderscoreToCamelCase" value="true" />
  15.     </settings>
  16. </configuration>

效果如图：

7.3 新建 config包，在其中新建配置类。共三个配置类，如图：

DataSourceConfiguration类的代码如下：
  1. package cn.stephen.study.demoproject.config.dao;
  2.  
  3. import com.mchange.v2.c3p0.ComboPooledDataSource;
  4. import org.springframework.beans.factory.annotation.Value;
  5. import org.springframework.context.annotation.Bean;
  6. import org.springframework.context.annotation.Configuration;
  7.  
  8. import java.beans.PropertyVetoException;
  9.  
  10. /**
  11.  * 数据库配置类
  12.  */
  13. @Configuration
  14. public class DataSourceConfiguration {
  15.  
  16.     @Value("${jdbc.driver}")
  17.     private String jdbcDriver;
  18.     @Value("${jdbc.url}")
  19.     private String jdbcUrl;
  20.     @Value("${jdbc.username}")
  21.     private String jdbcUsername;
  22.     @Value("${jdbc.password}")
  23.     private String jdbcPassword;
  24.  
  25.     @Bean(name = "dataSouce")
  26.     public ComboPooledDataSource createDataSouce() throws PropertyVetoException {
  27.         ComboPooledDataSource dataSource = new ComboPooledDataSource();
  28.         dataSource.setDriverClass(jdbcDriver);
  29.         dataSource.setJdbcUrl(jdbcUrl);
  30.         dataSource.setUser(jdbcUsername);
  31.         dataSource.setPassword(jdbcPassword);
  32.         //关闭连接后不自动commit
  33.         dataSource.setAutoCommitOnClose(false);
  34.         return dataSource;
  35.     }
  36. }
SessionFactoryConfiguration类的代码如下：
  1. package cn.stephen.study.demoproject.config.dao;
  2.  
  3. import org.mybatis.spring.SqlSessionFactoryBean;
  4. import org.springframework.beans.factory.annotation.Autowired;
  5. import org.springframework.beans.factory.annotation.Value;
  6. import org.springframework.context.annotation.Bean;
  7. import org.springframework.context.annotation.Configuration;
  8. import org.springframework.core.io.ClassPathResource;
  9. import org.springframework.core.io.support.PathMatchingResourcePatternResolver;
  10.  
  11. import javax.sql.DataSource;
  12. import java.io.IOException;
  13.  
  14. /**
  15.  * 数据库sqlSession配置类
  16.  */
  17.  
  18. @Configuration
  19. public class SessionFactoryConfiguration {
  20.  
  21.     @Value("${mapper_path}")
  22.     private String mapperPath;
  23.  
  24.     @Value("${mybatis_config_file}")
  25.     private String mybatisConfigFilePath;
  26.  
  27.     @Autowired
  28.     private DataSource dataSouce;
  29.     @Value("${entity_package}")
  30.     private String entityPackage;
  31.  
  32.     @Bean(name="sqlSessionFactory")
  33.     public SqlSessionFactoryBean createSqlSessionFactoryBean() throws IOException {
  34.         SqlSessionFactoryBean sqlSessionFactoryBean = new SqlSessionFactoryBean();
  35.         sqlSessionFactoryBean.setConfigLocation(new ClassPathResource(mybatisConfigFilePath));
  36.         PathMatchingResourcePatternResolver resolver = new PathMatchingResourcePatternResolver();
  37.         String packageSearchPath = PathMatchingResourcePatternResolver.CLASSPATH_ALL_URL_PREFIX+mapperPath;
  38.         sqlSessionFactoryBean.setMapperLocations(resolver.getResources(packageSearchPath));
  39.         sqlSessionFactoryBean.setDataSource(dataSouce);
  40.         sqlSessionFactoryBean.setTypeAliasesPackage(entityPackage);
  41.         return sqlSessionFactoryBean;
  42.     }
  43. }
TransactionManagementConfiguration类的代码如下：
  1. package cn.stephen.study.demoproject.config.service;
  2.  
  3. import org.springframework.beans.factory.annotation.Autowired;
  4. import org.springframework.context.annotation.Configuration;
  5. import org.springframework.jdbc.datasource.DataSourceTransactionManager;
  6. import org.springframework.transaction.PlatformTransactionManager;
  7. import org.springframework.transaction.annotation.EnableTransactionManagement;
  8. import org.springframework.transaction.annotation.TransactionManagementConfigurer;
  9.  
  10. import javax.sql.DataSource;
  11.  
  12. /**
  13.  * 事务配置类，不可缺少，尚未知具体作用
  14.  */
  15. @Configuration
  16. @EnableTransactionManagement
  17. public class TransactionManagementConfiguration implements TransactionManagementConfigurer{
  18.  
  19.     @Autowired
  20.     private DataSource dataSource;
  21.  
  22.     @Override
  23.     public PlatformTransactionManager annotationDrivenTransactionManager() {
  24.         return new DataSourceTransactionManager(dataSource);
  25.     }
  26. }

三个类的代码写完后，在DataSourceConfiguration类中会报找不到某些类的错，在下一步解决。

7.4 在 pom.xml文件的<dependencies></dependencies>插入以下两个依赖

  1. <!--线程池-->
  2. <dependency>
  3.     <groupId>com.mchange</groupId>
  4.     <artifactId>c3p0</artifactId>
  5.     <version>0.9.5.2</version>
  6. </dependency>
  7. <dependency>
  8.     <groupId>org.projectlombok</groupId>
  9.     <artifactId>lombok</artifactId>
  10.     <optional>true</optional>
  11. </dependency>

至此，项目配置已完成。。。。

8. 接下来，就按最普遍的分层代码目录结构来写一个基本的测试来测试项目是否能正常运行。代码目录结果如图：

一共需要新建4个包（controller、service、dao、entity）和一个目录（mapper）;

8.1 controller包放控制层代码，TestController类的代码如下：

  1. package cn.stephen.study.demoproject.controller;
  2.  
  3.

  7. import org.springframework.web.bind.annotation.PathVariable;
  8. import org.springframework.web.bind.annotation.RequestMapping;
  9. import org.springframework.web.bind.annotation.RequestMethod;
  10. import org.springframework.web.bind.annotation.RestController;
  11.  
  12. @RestController
  13. @RequestMapping("/demoproject/test")
  14. public class TestController {
  15.  
  16.     @Autowired
  17.     private TestService testService ;
  18.  
  19.     @RequestMapping(value = "/get/{id}",method = RequestMethod.GET)
  20.     public TestEntity test(@PathVariable Integer id){
  21.         System.out.println("id:" + id);
  22.         return testService.getById(id);
  23.     }
  24.  
  25. }

8.2 service层放业务处理层代码，TestService类的代码如下：

  1. package cn.stephen.study.demoproject.service;
  2.  
  3. import cn.stephen.study.demoproject.dao.TestDao;
  4. import cn.stephen.study.demoproject.entity.TestEntity;
  5. import org.springframework.beans.factory.annotation.Autowired;
  6. import org.springframework.stereotype.Service;
  7.  
  8. @Service
  9. public class TestService {
  10.  
  11.     @Autowired
  12.     private TestDao testDao ;
  13.  
  14.     public TestEntity getById(Integer id){
  15.         return testDao.getById(id);
  16.     }
  17. }

8.3 dao包放数据存取层代码，TestDao代码如下：

  1. package cn.stephen.study.demoproject.dao;

  3. import cn.stephen.study.demoproject.entity.TestEntity;

  5.  
  6. @Mapper
  7. public interface TestDao {
  8.  
  9.     TestEntity getById(Integer id);
  10.  
  11. }

对应的数据库表的sql语句如下：

  1. CREATE TABLE `test` (
  2.   `id` int(11) unsigned NOT NULL AUTO_INCREMENT,
  3.   `magic_id` varchar(32) NOT NULL,
  4.   `first_name` varchar(32) NOT NULL,
  5.   `last_name` varchar(32) NOT NULL,
  6.   PRIMARY KEY (`id`)

8.5 mapper目录放dao层代码的具体实现（这是mybatis的特色，用xml文件来实现数据存取），TestDaoMapper的内容如下：

  1. <?xml version="1.0" encoding="UTF-8" ?>
  2. <!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd" >
  3. <mapper namespace="cn.stephen.study.demoproject.dao.TestDao">
  4.     <!-- 根据主键查询-->
  5.     <select id="getById" resultType="cn.stephen.study.demoproject.entity.TestEntity" parameterType="java.lang.Integer" >
  6.         select  *
  7.         from test
  8.         where id = #{id}
  9.     </select>
  10. </mapper>
